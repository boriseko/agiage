АГЕНТЫ 2026 · СПРАВОЧНИК● 01 Что такое агент
Раздел 01

Что такое агент (и что — нет)

Два года назад ты открыл ChatGPT. Написал «напиши мне пост». Он написал. Ты скопировал. Ты вставил.

Это — чат-бот. Он отвечает на сообщения.

Агент — другое. Он не отвечает. Он делает.

Ты говоришь: «разбери эту папку из 40 PDF, найди те, где есть цифра 2026, сделай саммари и положи в Notion».

Чат-бот скажет, «вот как это сделать руками». Агент — сделает.

Цикл агента на человеческом

Внутри у агента простой цикл:

     ┌──────────┐     ┌──────────┐     ┌──────────┐
     │ ПОДУМАЛ  │ ──▶ │  СДЕЛАЛ  │ ──▶ │ ПРОВЕРИЛ │
     └──────────┘     └──────────┘     └────┬─────┘
          ▲                                  │
          │                                  ▼
          │                           ┌──────────┐
          └───────────────────────────│  ПОШЁЛ   │
                                      │  ДАЛЬШЕ  │
                                      └──────────┘
                   пока задача не закрыта

И так, пока задача не закрыта. Или пока он не упрётся и не попросит тебя помочь. Это всё. Никакой магии.

Внутри цикла агент делает две вещи, которых чат-бот не делает:

  • Сам режет задачу на шаги. Ты говоришь «собери отчёт по продажам». Он решает: вытащить данные → почистить → построить график → написать выводы → собрать в документ. Ты говоришь что. Он разбирается как.
  • Перечитывает себя и правит. Написал письмо — прочёл — нашёл, что тон слишком сухой — переписал. Без твоей команды.

Плюс две возможности сверху — инструменты (агент не только говорит, но и дёргает Gmail, SQL, браузер, файлы) и команда агентов (несколько агентов с ролями). Разберём их в разделе 05.

АГЕНТЫ 2026 · СПРАВОЧНИК● 01 Что такое агент

Чем агент отличается от пайплайна

Пайплайн — это «сделай A, потом B, потом C». Предсказуемо. Никакой свободы.

Агент — это «вот цель, вот инструменты, разберись сам». Свобода. Никакой предсказуемости.

◆ Правило

Начинай с пайплайна. К агенту переходи, только если задача ветвится и шаги предсказать нельзя.

90% того, что люди называют «мой AI-агент», — на самом деле пайплайн. И это нормально. Пайплайн дешевле, быстрее, надёжнее.

Осторожно: клей ≠ агент

Часто слышишь: «мой агент на n8n», «мой агент на Zapier».

Это не агенты. n8n, Make, Zapier — это клей. Они возят данные между сервисами по заранее прописанной схеме. Они не думают.

В паре работают прекрасно: клей таскает данные, агент думает.

◆ Пример

n8n поймал email → позвал Claude → Claude решил, что отвечать → n8n отправил через Gmail API.

Не путай. Если продукт обещает «AI-агенты», а внутри Zapier со стрелочками — ты купил автоматизацию, не агента.

АГЕНТЫ 2026 · СПРАВОЧНИК● 03 Код-агенты

Cursor и Windsurf — агрегаторы

Cursor и Windsurf стоят особняком. Они не привязаны к одной модели — дают выбирать между Claude, GPT-5, Gemini, DeepSeek. Переключай под задачу: креатив → Claude, код → GPT-5, дешёвое → DeepSeek.

Плюс у обоих есть своя моделька — Cursor-small, Windsurf Cascade Base — быстрые и дешёвые для лёгких задач.

◆ Фан-факт

Внутри Cursor и Windsurf можно запустить Claude Code — прямо в их интерфейсе. Больше пока никто так не умеет. Если уже сидишь в Cursor и боишься терминала — с Claude Code можно и не мучаться.

Что выбрать, если решаешь впервые

Если перед тобой чистый лист и надо выбрать один — Claude Code. Почему:

  • Экосистема. Модели Anthropic — чемпионы по сложным задачам. Claude — лучший дизайнер среди моделей: нужен красивый сайт или бот с адекватным UI — никто не даёт такого качества интерфейса из коробки.
  • Быстро обновляется. Sub-agents, skills, MCP-расширения — всё это Claude Code поддерживает первым. В блоге Anthropic раз в 2–3 недели новый апдейт.
  • Мультиагентность из коробки. Несколько агентов параллельно: один собирает данные, второй анализирует, третий пишет отчёт.
  • Комьюнити. Самая активная база примеров, шаблонов, MCP-серверов — именно вокруг Claude Code.
Ставим на одну лошадь — это Claude Code.

АГЕНТЫ 2026 · СПРАВОЧНИК● 03 Код-агенты

⚠ Памятка: блокировки Claude Code

▲ Важно знать

У Anthropic странная политика. Аккаунты иногда блокируют — без явной причины, независимо от того, что ты платишь. Человек пришёл пользоваться их же продуктом — получил бан.

Забанить могут на вход в Claude Code, на отдельный ключ API, на целую команду.

Это больная тема. Триггеры размыты — «подозрительная активность», «нарушение политик» — что значит каждый термин, Anthropic толком не раскрывает.

Как не попасть, что делать если попал, какие практики страхуют — проходим на интенсиве.

Для справочника: знай об этом заранее. Держи запасной вариант — Cursor + Codex — как план Б.

АГЕНТЫ 2026 · СПРАВОЧНИК● 04 Агентные системы

Немного истории

Год-два назад рынок клепал агентные системы под каждую нишу — саппорт, ресёрч, контент, аналитику, всё.

Это потому, что код-агенты были слабее. Не тянули длинный контекст, MCP ещё не существовало, мультиагентность не работала. И чтобы закрыть задачу под твои процессы — приходилось брать чужой SaaS.

Сейчас Claude Code закрывает 80% того, что эти продукты делали поодиночке. Вся та «экосистема» — максимально unusable. Инерция денег (инвесторы вложили раунды, команды пилят, маркетинг продаёт) её ещё держит — но для тебя смысл подключать чужой SaaS почти исчез.

Причин три

  1. Данные не твои. Отдаёшь письма, документы, клиентскую базу в чужой продукт. Поменяют цену, политику, закроются — ты в заложниках.
  2. Настраивать всё равно надо. Миф «открыл — работает» — это маркетинг. Неделя уйдёт на подгонку.
  3. Claude Code делает то же. За месяц-два практики соберёшь такой же инструмент под себя. Он твой, никто не отключит.

Исключения — та самая категория выше (сайты и приложения). Остальное собирается внутри Claude Code быстрее и лучше.

Маленькие агенты в своих системах

Отдельная важная мысль.

Тебе не всегда нужен Claude Code за $100. Для узкой задачи — сбор данных, анализ, систематизация, утренний отчёт — хватит голой API-подписки на OpenAI за $20/мес или копеечной open-source модели через OpenRouter.

◆ Пример

Агент каждое утро собирает данные с пяти источников (твоих каналов, блогов, конкурентов), анализирует, систематизирует, присылает тебе в Telegram дайджест. В 8 утра открываешь — готовый отчёт.

Ему не нужен Claude Opus за большие деньги. Хватит GPT-4o mini за копейки и 200 строк настройки.

Построй сам

Бывает — готового мало. Надо своё.

Например: агент, который каждое утро читает твою почту, сверяет письма с CRM, пишет черновики ответов и кладёт их в Gmail. Такого коробочного продукта нет. Собрать — можно.

└───────────────┘    └──────────────────┘    └─────────────────┘

Мозг

LLM-провайдер. Думалка агента. Подключается по API одной строчкой.

  • Anthropic Claude — креатив, код, длинный контекст, лучший дизайн.
  • OpenAI GPT-5 — самый «ровный», хорош везде, но дорог.
  • Google Gemini — картинки, мультимодальность, большой контекст.
  • DeepSeek — дёшево, умно, китайский.
  • Qwen — ещё один китайский, open-source.

Все модели примерно на одном уровне, дальше — под задачу. Для креатива и дизайна — Claude. Для дешевизны — DeepSeek или Qwen. Для картинок и видео — Gemini.

Переключать мозг можно одной строчкой в конфиге. Это даёт страховку: провайдер упал / подорожал / забанил — переключил.

АГЕНТЫ 2026 · СПРАВОЧНИК● 05 Построй сам

Руки

Чтобы мозг не только думал, но и действовал. Три способа дать агенту руки.

MCP (Model Context Protocol)

USB-C для агентов. Один раз воткнул — работает везде.

  Один раз настроил MCP-сервер →
  работает в Claude Code, Cursor, ChatGPT, Windsurf:

  ┌────────────┐                    ┌──────────────┐
  │   Notion   │◀── MCP-сервер ──▶ │  Claude Code │
  └────────────┘                    ├──────────────┤
                                    │    Cursor    │
  ┌────────────┐                    ├──────────────┤
  │   Gmail    │◀── MCP-сервер ──▶ │   ChatGPT    │
  └────────────┘                    ├──────────────┤
                                    │   Windsurf   │
  ┌────────────┐                    └──────────────┘
  │   GitHub   │◀── MCP-сервер ──▶
  └────────────┘

Готовых MCP-коннекторов уже сотни: Gmail, Calendar, базы данных, Figma, GitHub, Telegram, Slack, Notion, Linear, Jira.

Function calling

Ты описываешь конкретное действие — «вот как залезть в нашу CRM» — агент дёргает его по команде. Технология старше и проще MCP.

Computer use

Агент видит экран и двигает мышкой, как человек. Полезен там, где нет API — только интерфейс.

▲ Важный нюанс

Computer use — это свойство самой модели, не внешняя штука. Если у модели нет computer use — ты не можешь его «подключить». Сейчас computer use полноценно поддерживают только Claude Sonnet 4.6+ и (с ограничениями) GPT-4o.

  │   живая                 │     │   сама база             │
  └─────────────────────────┘     └─────────────────────────┘

Короткая. То, что помещается в один запрос — контекстное окно. У Claude Sonnet 4.6 — 200K токенов (~400 страниц). У Opus 4.7 1M — миллион (~2000 страниц).

Долгая. Во внешней базе. Векторная база данных — библиотека, где книги расставлены не по алфавиту, а по смыслу.

◆ Пример

Ты скидываешь агенту 5 000 сообщений из рабочего чата. Он раскладывает их по смыслу. Спрашиваешь: «что я решил про клиента Х три месяца назад?» — достаёт релевантные сообщения, даже если ты не помнишь ни слова из той переписки.

Это и есть RAG (Retrieval-Augmented Generation). Сервисы векторных БД: Pinecone, Weaviate, Chroma, Qdrant.

АГЕНТЫ 2026 · СПРАВОЧНИК● 05 Построй сам

Как собрать на человеческом

Возьми задачу «агент-ассистент в Telegram».

  1. Мозг: Claude Sonnet 4.6.
  2. Руки: MCP Telegram + MCP Notion + MCP Gmail.
  3. Память: одна векторная БД на личные заметки.
  4. Команда: один агент. Мультиагентность тут избыточна.

Собирается за вечер. Работает — годами.

Важное

Всё, что описано выше — уже внутри Claude Code. MCP из коробки. Sub-agents для команды. Skills для долгой памяти. Function calling из коробки.

То есть «построй сам» для 95% задач — это не пилить с нуля Python. Это открыть Claude Code и сказать: «сделай мне агента, который [читает почту / сверяет CRM / пишет черновики / кладёт в Gmail]».

Всё остальное — внешние фреймворки, n8n-связки, самописные LangGraph — нужно в тех 5% кейсов, когда задача действительно особенная.

В 2026 году «построй сам» ≈ Claude Code.

Три правила, которые спасают от всех семи

  1. Human-in-the-loop на критичном. Отправка письма, списание денег, удаление файла, публичное сообщение — всегда финальное подтверждение человеком.
  2. Ограничители заранее. Правила: что нельзя, какие суммы, какие темы. Лучше заложить сразу, чем чинить потом.
  3. Регулярные проверки. Раз в неделю прогоняй агента по 20–50 типичным сценариям. Деградировал — ловишь сразу, не от клиентов.
Агент без этих трёх — бомба замедленного действия.

Неважно, соло ты работаешь или в компании на 10 000 человек.

АГЕНТЫ 2026 · СПРАВОЧНИК● 08 Оплата из РФ
Раздел 08

Как оплатить из РФ

Карта российского банка на OpenAI / Anthropic / Cursor не проходит. Решения есть — всё легально.

1
OpenRouter

Один агрегатор — сотни моделей (GPT, Claude, Gemini, DeepSeek, Qwen). Платишь кредитами, пополняешь рублями через крипту. Claude Code умеет работать через OpenRouter.

2
Российские реселлеры

ProxyAPI, Vsegpt, GPTunnel. Отдают ключи OpenAI и Anthropic, пополняешь через СБП, отдают токены один-в-один.

3
Крипта → зарубежная карта

Bitget, Binance Pay выпускают карты под USDT. Оплачиваешь любую подписку: ChatGPT Plus, Claude Pro, Cursor, Lovable.

4
Виртуальные карты

Pyypl, Wallester, Zen, QForm. Получаешь реквизиты, пополняешь рублями, платишь подписки.

◆ Что выбрать

Строишь агента на API → OpenRouter или российский реселлер.
Нужна подписка (ChatGPT Plus / Claude Pro / Cursor) → крипта или виртуалка.

Подробнее и с пошаговыми инструкциями — на практикуме. Главное: оплата решаема. Это не стоп-фактор.

Раздел 09

Что дальше

Ты дочитал. Это уже немало — многие и этого не делают. Теперь в голове есть карта.

Но справочник — это карта. Маршрут собирается руками.

Карта показывает, что бывает. Маршрут — твой путь через это: с конкретным агентом, конкретной задачей и человеком рядом, когда упрёшься в стену (а ты упрёшься — так у всех).

Приходи на практикум «ВАЙБ»

Семь вечеров. Собираешь свой проект — не демку для галочки, а то, что нужно тебе завтра. Что мы делаем:

  1. Разбираемся, что такое агент и что с ним можно — через твои задачи, не абстрактные примеры.
  2. Ставим Claude Code (или выбираешь другой агент) и настраиваем под твой сценарий.
  3. Учимся ставить задачу так, чтобы агент понял с первого раза.
  4. Подключаем MCP — почту, заметки, диск, мессенджеры, CRM, календари.
  5. Разворачиваем проект на сервере — домен, хостинг, SSL. Работает 24/7 без тебя.
  6. Собираем живой продукт в прямом эфире с группой — сайт, Telegram-бот или автоматизацию.
  7. Разбираемся, где агент врёт и где его нельзя оставлять одного.
  8. Защищаем данные и деньги — что никогда не даём агенту, какие правила ставим.
  9. Отдельно: как не попасть под блокировку Claude Code — что триггерит бан, как вести аккаунт.
  10. Попадаем в закрытое комьюнити тех, кто прошёл это вместе с тобой.
